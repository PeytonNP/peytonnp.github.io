Projects

This project is a mobile application that uses the powerful healthcare related APIs that are becoming more relevant. This project initially became a spin-off of a project I worked on during a hackathon. That project was an Apple Watch application to monitor attributes during a biking session (This was also one of the first times I attempted iOS development). I've always been interested in applying technology to healthcare. Witnessing a lot of college students with large amounts of stress / anxiety coupled with my interest in Machine Learning, I wanted to make an application that would first monitor then learn and predict anxiety attacks / high levels of stress / identify stressors. This project is becoming a lot more complicated than I originally thought because of user experience (users likely won't keep an app open consistently / constant user input and feedback), data storage (what information to collect and how to store it), accuracy (false positives for health conditions), complexity (how can I pull accurate medical data and compare it to the data the user is generating), medical understanding (I don't have a medical background), and rights of users (user information and privacy). The biggest challenge I've had has to deal with my limited / ongoing learning of Machine Learning. This semester allows me to dive more into the topic inside the classroom in addition to my own research to try to understand what is the best way I can apply it to this project. If all goes well with the application focused on anxiety, maybe similar logic can be applied to other conditions.

This project has been my longest running project as it has seen many, many changes and updates. The idea was to build a robot that would automatically create custom light patterns that could then be photographed when light painting. What is light painting? I began this project during my freshman year of University as a way to keep exploring robotics, after I graduated from my high school team. This project included the design, construction, and programming of a robot. The inital plan was to create a simple robot with an arm attached to a four-wheel drive base. It was designed to be operated via a game controller (Playstation or Xbox Controller), similar to the way our high school team controlled them. The original design included 2-4 DC motors controlled via an Arduino Uno. After ordering parts and materials, I noticed that I should've performed some calculations for the motors. After realizing the motors would not be precise enough to achieve accurate, repititive motions. At this point, it was back to design. I wanted to create a way to make the motors work for my needs so I didn't need to purchase new materials. So I decided that to extensively test the motors and use them only for the base of the robot. I decided to purchase a servo and use that for the single arm, as the movements of the arm would be the most important. As this change was being implemented, I was still using the Arduino Uno. In the tech world, the buzz became IoT and making everything wireless. I decided to make yet another change by switching to a Raspberry Pi 3 as the device would "future-proof" the design. In doing that, I would be able to wireless control the robot via bluetooth, create a mobile application (if needed; as another means to control the robot), provide a new way to connect to other technologies, and various new add on's. Switching to a Raspberry Pi has been the latest major hardware change and I am currently in the testing process. In the future, I may decide to switch all of the mechanical hardware to stepper motors for precision and keep a digital log of any measurements.
